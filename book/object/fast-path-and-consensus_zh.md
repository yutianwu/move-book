# 快速路径与共识

对象模型允许可变的交易执行路径，这取决于对象的所有权类型。交易执行路径决定了网络如何处理和验证交易。在本节中，我们将探讨Sui中不同的交易执行路径以及它们如何与共识机制交互。

## 并发挑战

在其核心，区块链技术面临着一个基本的并发挑战：在去中心化环境中，多方可能试图同时修改或访问相同的数据。这需要一个用于排序和验证交易的系统来支持网络的一致性。Sui通过共识机制解决了这一挑战，确保所有节点就交易的顺序和状态达成一致。

考虑一个市场场景，Alice和Bob同时尝试购买同一资产。网络必须解决这一冲突以防止双重支付，确保最多一个交易成功，而另一个被正当拒绝。

## 快速路径

然而，并非所有交易都需要相同级别的验证和共识。例如，如果Alice想要将她拥有的对象转移给Bob，网络可以处理这个交易而不需要相对于网络中所有其他交易对其进行排序，因为只有Alice有权访问该对象。这被称为_快速路径_执行，其中访问账户拥有对象的交易被快速处理，无需广泛的共识。没有并发数据访问 -> 更简单的挑战 -> 快速路径。

另一个允许快速路径执行的所有权模型是_不可变状态_。由于不可变对象不能改变，涉及它们的交易可以快速处理而无需排序。

## 共识路径

确实访问共享状态的交易——在Sui上用共享对象表示——需要排序以确保状态在所有节点上一致更新。这被称为通过_共识_执行，其中访问共享对象的交易需要经过协议过程以维护网络一致性。

## 对象拥有的对象

最后，重要的是要提到由其他对象拥有的对象遵循与父对象相同的规则。如果父对象是_共享的_，子对象也是传递性共享的。如果父对象是不可变的，子对象也是不可变的。

## 总结

- **快速路径：** 涉及账户拥有对象或不可变共享状态的交易被快速处理，无需广泛的共识。
- **共识路径：** 涉及共享对象的交易需要排序和共识以确保网络完整性。
- **对象拥有的对象：** 子对象继承父对象的所有权模型。
